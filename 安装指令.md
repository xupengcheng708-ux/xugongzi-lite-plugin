# 许公子工具包 · 安装指令

> **学员使用**：把下面这段话**整段复制**，粘到你的 Claude Code 对话框（VSCode 扩展里），Claude 会自动帮你装好。
> 安装过程中 Claude 会问你几个问题（保存路径、是否允许执行命令），回答就行。

---

```
我要安装「许公子工具包」—— 许公子分享的抖音内容创作者 Claude Code 工具包。请按以下步骤自动执行，每步简短告诉我进度：

## 1. 下载工具本体

执行：git clone https://github.com/xupengcheng708-ux/xugongzi-lite-plugin.git ~/.claude/plugins/xugongzi-lite

如果目录已存在，进入目录跑 git pull 更新到最新。
如果 git 命令不存在，告诉我跑 `xcode-select --install` 装，然后停下等我。

## 2. 把 skill 链接到 ~/.claude/skills/

执行：
```
mkdir -p ~/.claude/skills
for name in 灵感提取 对标拆解 抓整个号 任务状态; do
    ln -sfn ~/.claude/plugins/xugongzi-lite/skills/$name ~/.claude/skills/$name
done
```

## 3. 问我 4 个配置问题（一次问一个，等我回答再问下一个）

### 3.1 工作路线
问我：「选哪条工作路线？」给选项：
- A = 满血版（Apple Silicon Mac M1/M2/M3/M4 专属，转写快 3-5 倍，抖音主页可按点赞筛）
- B = 通用版（跨平台，推荐新手）
默认 B。

### 3.2 灵感池目录
问我：「/灵感提取 产出的 md 存哪？」默认 `~/Desktop/许公子工具包/灵感池`

### 3.3 拆解池目录
问我：「/对标拆解 产出的 md 存哪？」默认 `~/Desktop/许公子工具包/拆解池`

### 3.4 抓账号目录
问我：「/抓整个号 批量归档存哪？」默认 `~/Desktop/许公子工具包/抓账号`

问完后：
- 每个路径的 ~ 展开成绝对路径
- mkdir -p 建好目录
- 写入 ~/.xugongzi-toolkit/config.json：
  {
    "mode": "a 或 b",
    "inspiration_dir": "<绝对路径>",
    "review_dir": "<绝对路径>",
    "account_dir": "<绝对路径>",
    "updated_at": "<当前 UTC ISO 时间>",
    "source": "install-script"
  }

## 4. 检测系统依赖

检查这些命令有没有装：
- 必需：jq、ffmpeg、yt-dlp、python3
- 路线 A 额外：mlx_whisper
- 路线 B 额外：python3 -c "import whisper" 不报错（openai-whisper 包）

缺哪个列出来 + 给对应安装命令：
- jq / ffmpeg / yt-dlp → brew install xxx
- mlx-whisper → pip3 install mlx-whisper
- openai-whisper → pip3 install openai-whisper

问我：「要我帮你跑这些安装命令吗？」我答应才跑（因为 brew/pip 会下东西）。

## 5. 问我要不要装 Raycast 快捷键（可选，推荐）

问我：「你装了 Raycast 吗？要装快捷键入口吗？（⌥+空格 搜「灵感提取」→ 回车 → 填 URL → 提交，跟许公子演示的一模一样）」

- 我说「没装 Raycast / 不要 / 跳过」→ 跳到 Step 6
- 我说「要」→ 按下面流程执行：

### 5.1 检查 Node.js（Raycast Extension 需要）

```
node --version
```

如果报 `command not found` 或版本 < 18：
- 问我：「要帮你装 Node.js 吗？（`brew install node`，大约 1 分钟）」
- 我同意 → 跑 `brew install node`

### 5.2 装 Extension 依赖

```
cd ~/.claude/plugins/xugongzi-lite/raycast-extension
npm install
```

这步 1-3 分钟，告诉我进度。

### 5.3 注册 Extension 到 Raycast

⚠️ **特殊操作**：`npm run dev` 会启动一个持续运行的开发服务器。我们只需要它**运行一次 + 让 Raycast 识别 extension**，然后就能关掉。

执行：

```
cd ~/.claude/plugins/xugongzi-lite/raycast-extension
npm run dev &
RAY_PID=$!
# 等 15 秒让 Raycast 注册 extension
sleep 15
# 杀掉 dev server
kill $RAY_PID 2>/dev/null
```

告诉我「Extension 已注册到 Raycast。dev server 已关闭，extension 仍然在 Raycast 里。」

### 5.4 验证

告诉我：「按 ⌥+空格 呼出 Raycast → 搜「灵感提取」→ 应该看到 4 个紫色方块 Command（灵感提取 / 对标拆解 / 抓整个号 / 任务状态）」

## 6. 完成通告

全部搞完后告诉我：

✅ 安装完成
  · 工具装在: ~/.claude/plugins/xugongzi-lite/
  · 配置文件: ~/.xugongzi-toolkit/config.json
  · 4 个 skill 已链接到 ~/.claude/skills/
  · Raycast 快捷键（如选装）：⌥+空格 → 4 个命令

⚠️ 必做的最后一步：
  · Cmd+Q 完全退出 VSCode（不是关窗口）
  · 重新打开 VSCode → skill 才会加载
  · 重开后试试：跟我说「灵感提取 <任意抖音链接>」，或按 ⌥+空格 搜"灵感提取"

遇到任何错误停下来告诉我，不要硬扛。
```
