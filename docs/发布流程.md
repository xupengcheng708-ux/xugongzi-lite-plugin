# 发布流程（许公子自己看）

## 首次发布（只做一次）

### 1. 占位符替换

repo 里有两处 `xupengcheng708-ux` 占位符（README.md 和 quick-install.sh），一次性替换成你的 GitHub 用户名：

```bash
cd ~/Documents/xugongzi-lite-plugin
grep -rl 'xupengcheng708-ux' . | xargs sed -i '' 's/xupengcheng708-ux/你的GitHub用户名/g'
```

**验证**：

```bash
grep -r 'xupengcheng708-ux' . || echo "✅ 已全部替换"
```

### 2. 本地自测

先用 symlink 模拟学员装 plugin，自己跑一次端到端：

```bash
mkdir -p ~/.claude/plugins
ln -sf ~/Documents/xugongzi-lite-plugin ~/.claude/plugins/xugongzi-lite
```

打开 Claude Code（任意目录），依次试：

- [ ] `/xugongzi-init` → 跑通向导，config.json 生成
- [ ] `/灵感提取 <一条抖音URL>` → 能后台启动
- [ ] `/任务状态` → 能看到任务
- [ ] 任务完成后，md 文件出现在配置的灵感池目录
- [ ] `/对标拆解 <URL>` → 文案 + 4 段分析都有
- [ ] `/抓整个号 <主页URL>` → 列出作品清单，让你选

有问题改完再继续下一步。

### 3. Push 到 GitHub

```bash
cd ~/Documents/xugongzi-lite-plugin
git init
git add .
git commit -m "初版：灵感提取 / 对标拆解 / 抓整个号 / 任务状态"
```

GitHub 网页上建一个**公开**的空 repo，名字 `xugongzi-lite-plugin`（**不要勾 "Add README"、"Add .gitignore"、"Add license"**，保持空的）。

```bash
git branch -M main
git remote add origin git@github.com:你的GitHub用户名/xugongzi-lite-plugin.git
git push -u origin main
```

### 4. 发给学员的文案

复制这一段发到学员群：

> **许公子工具包 v0.1 发布了**
>
> 开 Terminal（终端），把下面这一整行粘进去，回车：
>
> ```
> curl -fsSL https://raw.githubusercontent.com/你的GitHub用户名/xugongzi-lite-plugin/main/quick-install.sh | bash
> ```
>
> 装完按提示做 3 步就能用。
>
> 前提是你已经装好 Claude Code（还没装的先看《Claude Code 安装教程》）。
>
> 问题在群里 @ 我，带截图。

## 学员常见问题（FAQ 备忘）

整理好之后可以贴到 GitHub repo 的 Issues 或 Discussions。

| 症状 | 原因 | 解决 |
|---|---|---|
| `/xugongzi-init` 命令识别不到 | Claude Code 没完全重启 | Cmd+Q 退出后重新打开 |
| `check_deps.sh: command not found` | plugin 路径不对 | `ls ~/.claude/plugins/xugongzi-lite/scripts/lib/` 确认文件在 |
| 抖音下载失败 | cookie 过期（路线 A）| 重开 DouK 登一次 |
| whisper 报错 | 模型没下全 | 手动跑一次 `whisper --help` 等自动下载 |
| mlx_whisper not found | 不是 Apple Silicon / 没装 | 换路线 B 或 `pip3 install mlx-whisper` |
| 文件存到奇怪位置 | 初始化时路径填错 | 重跑 `/xugongzi-init` 重填 |
