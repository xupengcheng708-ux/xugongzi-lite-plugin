# 任务状态

读 `~/.xugongzi-toolkit/tasks.log` 显示所有任务（正在跑的 + 已完成的近 20 条）。

## 执行步骤

```bash
bash ${CLAUDE_PLUGIN_ROOT:-$HOME/.claude/plugins/xugongzi-lite}/scripts/lib/task_log.sh list
```

脚本会输出类似：

```
状态     任务ID        类型          进度  参数                   开始时间
🟢 跑中  xgz-2026...   灵感提取      60%   https://v.douyin...   14:32:10
🟢 跑中  xgz-2026...   抓整个号      8/10  余震彦的主页          14:28:45
✅ 完成  xgz-2026...   对标拆解      100%  https://v.douyin...   14:20:03
❌ 失败  xgz-2026...   灵感提取      -     https://xxx...        14:15:22  (日志: ~/.xugongzi-toolkit/logs/xxx.log)
```

## 回复学员

原样展示表格。如果有失败任务，额外提示一句：
「有失败的任务，要看详情吗？」学员说是 → `cat` 对应日志文件展示。

如果所有任务都完成了，告诉学员："当前没有正在跑的任务。"

## 参数（可选）

- `/任务状态 clean` → 清理日志中 24 小时前的完成任务（调 `task_log.sh clean`）
- `/任务状态 <任务ID>` → 只看某个任务的详细日志
