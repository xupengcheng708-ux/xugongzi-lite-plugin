# 对标拆解

接收单条对标视频 URL，下载 → 转写 → 跑「钩子/结构/金句/可借鉴点」分析 → 生成 md 存到学员配置的「拆解池」目录。

## 执行步骤

### 1. 解析参数

用户输入：`$ARGUMENTS`。提取 URL，同 `/灵感提取`。

### 2. 读配置

```bash
bash ${CLAUDE_PLUGIN_ROOT:-$HOME/.claude/plugins/xugongzi-lite}/scripts/lib/config.sh get mode
bash ${CLAUDE_PLUGIN_ROOT:-$HOME/.claude/plugins/xugongzi-lite}/scripts/lib/config.sh get review_dir
```

### 3. 下载+转写（后台任务）

**路线 A**：
```bash
TASK_ID=$(bash ${CLAUDE_PLUGIN_ROOT:-$HOME/.claude/plugins/xugongzi-lite}/scripts/lib/task_log.sh create "对标拆解-抽文案" "<URL>")
nohup bash ${CLAUDE_PLUGIN_ROOT:-$HOME/.claude/plugins/xugongzi-lite}/scripts/mode_a/inspiration.sh "$TASK_ID" "<URL>" "--target-dir=<review_dir>" > /dev/null 2>&1 &
```

**路线 B**：
```bash
TASK_ID=$(bash ${CLAUDE_PLUGIN_ROOT:-$HOME/.claude/plugins/xugongzi-lite}/scripts/lib/task_log.sh create "对标拆解-抽文案" "<URL>")
nohup bash ${CLAUDE_PLUGIN_ROOT:-$HOME/.claude/plugins/xugongzi-lite}/scripts/mode_b/inspiration.sh "$TASK_ID" "<URL>" "--target-dir=<review_dir>" > /dev/null 2>&1 &
```

### 4. 等后台完成 + 跑分析

**与 `/灵感提取` 不同**：拆解命令要等文案抽完才能做分析。

两种做法：

**做法 A（推荐）：异步 + 轮询**
- 立即告诉学员"已启动下载，预计 N 秒后回来做分析"
- 用 `bash ${CLAUDE_PLUGIN_ROOT:-$HOME/.claude/plugins/xugongzi-lite}/scripts/lib/task_log.sh wait <TASK_ID> --timeout 600` 阻塞等完成
- 完成后读 md 文件跑分析

**做法 B：学员手动触发**
- 只做下载+转写（跟灵感提取一样）
- 告诉学员："文案抽完了，我再帮你分析吗？"
- 学员说"是" → 读 md 跑分析

默认走做法 A（一气呵成，不用学员额外点）。

### 5. 分析四个维度

读完文案后，你直接写分析，追加到同一个 md 文件末尾：

```markdown
---

## 🎯 拆解分析

### 钩子（开场 3 秒）
<前 15-25 字的具体内容 + 用了什么勾人技巧（痛点/好奇/承诺/反常识等）+ 为什么能让人停下来看>

### 结构（整体框架）
<整体叙事逻辑是什么（问题→解答 / 故事→转折 / 列举→总结 等）+ 节奏怎么安排 + 结尾用了什么收场>

### 金句（原文引用）
<选 1-3 句最值得借鉴的话，每句说明好在哪里（语言特点 / 心理触发点）>

### 可借鉴的点
<具体的、可以直接复用的 1-3 个点，不是泛泛"学习他的风格">
```

### 6. 质量红线

- **金句必须原文引用**，不要改写
- **可借鉴点要具体**（"用他的反转钩子模板做 XX 选题"，不是"学他的风格"）
- **不要打分**、不要写学术化分析（"本案例体现了 UGC 特征"）
- 每个推断尽量带证据（引用文案里某句话）

### 7. 告诉学员

```
✅ 拆解完成：<视频标题>
  文件：<review_dir>/YYYY-MM-DD-<标题>.md
  四维分析已写入
```
